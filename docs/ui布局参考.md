- **好感度条**：显示在顶部信息栏，取值 -100~100，数值变化时有动画过渡
- **点击头像**：弹出人物信息弹窗（ProfileDialog）
- **点击“⋮”菜单**：可操作“编辑联系人”、“删除聊天记录”、“导出聊天记录”等（后期扩展）

## 4. 输入面板 (InputPanel)

text

```
┌──────────────────────────────────────────────────────────────┐
│  [我是谁? ▼]  [输入消息... (多行)]                           │
│                                                             │  ← 消息输入区
│  ┌─────────────────────────────────────────────────────────┐ │
│  │  😊  [📎 图片/表情描述]                                │ │  ← 工具栏
│  └─────────────────────────────────────────────────────────┘ │
│                                                             │
│  ┌─────────────────────────────────────────────────────────┐ │
│  │  建议1: [选择]  建议2: [选择]  建议3: [选择]  [⟳ 刷新] │ │  ← 建议按钮区
│  └─────────────────────────────────────────────────────────┘ │
│                                                             │
│  [策略调整 🧠]                          [发送 →]             │  ← 底部操作栏
└──────────────────────────────────────────────────────────────┘
```



### 组件说明

| 组件           | 说明                                                         |
| :------------- | :----------------------------------------------------------- |
| **发言人切换** | `QComboBox`，选项：“我 (user)” / “对方 (target)”，用于标记当前输入属于谁 |
| **消息输入框** | `QTextEdit`，支持多行输入    |
| **表情/附件**  | 预留按钮，当前版本仅支持文本输入；图片/表情由用户手动描述文字 |
| **建议按钮区** | 召唤智能体后，动态生成 3 个选择按钮，点击即填入输入框或直接发送 |
| **刷新按钮**   | 重新生成 3 条建议（需消耗 Token，二次确认）                  |
| **策略调整**   | 常驻按钮，点击弹出策略调整弹窗                               |
| **发送按钮**   | 将输入框内容作为当前发言人消息发送                           |

### 交互逻辑

- **切换发言人** → 不影响已发送的消息，仅影响下一条消息的 sender_type
- **发送消息** → 保存到数据库 → 追加到聊天区 → 清空输入框
- **选择建议** → 将选中的建议文本填入输入框（或直接发送，用户可在设置中选择行为）
- **刷新建议** → 重新调用 LLM，生成新的 3 条建议（覆盖之前的）

## 9. 静默/常开模式切换

模式切换不在独立弹窗中，而是作为主窗口的顶部或底部状态控件：

text

```
┌──────────────────────────────────────────────────────────────┐
│  💬 恋爱军师                              [🔇 静默] [🔊 常开] │  ← 顶部右侧
└──────────────────────────────────────────────────────────────┘
```



### 交互逻辑

- **静默模式（默认）**：智能体不主动发言，用户点击“召唤”才生成建议
- **常开模式**：智能体在后台持续分析对话，当检测到冷场或情绪波动时，通过系统通知或 UI 角标提醒用户
- 切换模式时无需重启应用，立即生效

## 设计原则

### 颜色方案

| 用途           | 色值      | 说明                         |
| :------------- | :-------- | :--------------------------- |
| 主色（微信绿） | `#07C160` | 发送按钮、选中状态、我方气泡 |
| 辅色           | `#1AAD19` | 悬停状态、链接               |
| 背景色         | `#EDEDED` | 主窗口背景                   |
| 卡片/气泡      | `#FFFFFF` | 对方消息气泡、弹窗背景       |
| 分割线         | `#D9D9D9` | 列表分割线                   |
| 文字主色       | `#333333` | 标题、正文                   |
| 文字辅色       | `#999999` | 时间戳、预览文字             |
| 危险/负值      | `#FA5151` | 好感度负值                   |
| 警告/提示      | `#FF9F00` | 策略调整弹窗边框             |

### 字体规范

| 用途        | 字号 | 字重    |
| :---------- | :--- | :------ |
| 联系人名称  | 14px | Medium  |
| 消息正文    | 14px | Regular |
| 时间戳/预览 | 12px | Regular |
