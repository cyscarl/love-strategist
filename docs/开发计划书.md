# 聊天辅助工具 - Python/PyQt5 开发计划书

## 背景

开发一款基于 PyQt5 的桌面应用，通过大模型 API 驱动智能体，辅助用户分析与目标对象的聊天记录，生成高情商回复建议，帮助用户推进人际关系。核心功能包括：联系人管理、聊天记录存储、人物画像自动分析、AI 回复建议生成（类似 Galgame 的多选交互）。

> **项目代码根目录**：`code/`，所有源码、测试、脚本位于此目录下。`docs/`、`config/`、`data/`、`temp/` 位于项目根目录（与 `code/` 同级）。

------

## 数据库策略：SQLite 单文件

**核心需求**：纯本地应用，单用户使用，零配置启动。所有数据（联系人、消息、人物画像、Skill 日志）存储在同一 SQLite 数据库文件中。

| 场景             | 数据库          | 说明                                               |
| :--------------- | :-------------- | :------------------------------------------------- |
| 本地单机（默认） | SQLite          | 单文件 `data/love_strategist.db`，无需安装额外服务 |
| 数据备份         | SQLite 文件复制 | 用户可直接复制 .db 文件进行备份或迁移              |

SQLite 足以支撑数千条聊天记录的性能需求，且无需额外的数据库驱动。`sqlite3` 由 Python 标准库内置，零额外依赖。

## 2. 数据库设计

### 2.1 通用说明

使用 SQLite 单文件数据库 `data/love_strategist.db`，由原生 `sqlite3` + 手写 SQL 管理。所有表通过 `database.py` 统一创建和连接。

**连接方式**：`sqlite:///data/love_strategist.db`，启用 `check_same_thread=False` 以支持多线程读操作（写操作使用 `threading.Lock` 保护）。

### 2.2 表结构

```sql
-- 联系人表
CREATE TABLE contacts (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT NOT NULL,
    avatar TEXT,                        -- 头像路径（本地文件路径或 base64）
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 消息表
CREATE TABLE messages (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    contact_id INTEGER NOT NULL,
    sender_type TEXT NOT NULL,          -- 'user' | 'target'
    content TEXT NOT NULL,
    content_type TEXT DEFAULT 'text',   -- 'text' | 'image' | 'emoji'
    timestamp TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    is_edited BOOLEAN DEFAULT 0,
    FOREIGN KEY (contact_id) REFERENCES contacts(id) ON DELETE CASCADE
);
CREATE INDEX idx_messages_contact_time ON messages(contact_id, timestamp DESC);

-- 人物画像表
CREATE TABLE profiles (
    contact_id INTEGER PRIMARY KEY,
    basic_info TEXT,                    -- JSON: {"age": 25, "occupation": "设计师", "city": "上海"}
    personality TEXT,                   -- JSON: {"traits": ["外向", "细心"], "tags": ["社恐", "慢热"]}
    hobbies TEXT,                       -- JSON: ["摄影", "旅行", "看电影"]
    behavior_patterns TEXT,             -- JSON: {"reply_style": "慢", "active_time": "晚上"}
    affinity_score INTEGER DEFAULT 0,   -- -100 ~ 100
    summary TEXT,                       -- 精简摘要（供 LLM 使用的文字描述）
    last_updated TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (contact_id) REFERENCES contacts(id) ON DELETE CASCADE
);

-- Skill 执行日志（可选，用于调试和 Token 统计）
CREATE TABLE skill_logs (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    skill_name TEXT NOT NULL,
    contact_id INTEGER,
    input_summary TEXT,                 -- 输入上下文摘要
    output_summary TEXT,                -- 输出结果摘要
    token_used INTEGER,
    duration_ms INTEGER,                -- 执行耗时（毫秒）
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (contact_id) REFERENCES contacts(id) ON DELETE SET NULL
);
```

### 2.3 初始数据

首次启动时自动创建空数据库，无预设数据。用户通过 UI 手动创建第一个联系人。

### 2.4 数据迁移

暂无迁移需求。未来版本升级时在 `code/src/storage/migrations/` 目录下添加迁移脚本。

## 3. 模块职责与关键文件

### 3.1 Storage 层 (`code/src/storage/`)

- `database.py`：管理 SQLite 连接，提供 `get_connection()`、`execute_query()`、`execute_write()` 接口，写操作使用 `threading.Lock` 保护。
- `contact_dao.py`：联系人 CRUD（`create`、`get_by_id`、`get_all`、`update`、`delete`）。
- `message_dao.py`：消息 CRUD + 分页查询（`get_recent(contact_id, limit)`、`get_before(contact_id, before_timestamp, limit)`、`create`、`update`、`delete`）。
- `profile_dao.py`：画像 CRUD（`get_by_contact`、`create_or_update`）。

### 3.2 Models 层 (`code/src/models/`)

纯数据类（使用 `@dataclass` 或普通类），属性名使用 snake_case，与数据库字段对应。

- `contact.py`：`Contact(id, name, avatar, created_at, updated_at)`
- `message.py`：`Message(id, contact_id, sender_type, content, content_type, timestamp, is_edited)`
- `profile.py`：`Profile(contact_id, basic_info, personality, hobbies, behavior_patterns, affinity_score, summary, last_updated)` — 其中 JSON 字段使用 `dict`/`list` 类型，由 DAO 层负责序列化/反序列化。

### 3.3 Services 层 (`code/src/services/`)

- `llm_service.py`：封装大模型 API 调用，支持 OpenAI 兼容接口。提供 `chat_completion(messages, temperature)` 方法，返回 `(response_text, token_usage)`。所有调用在 `QThread` 中异步执行。
- `profile_service.py`：调用 `ProfileAnalyzer` Skill 分析聊天记录，生成/更新人物画像。提供 `update_profile(contact_id)`、`get_profile_summary(contact_id)` 方法。
- `context_summarizer.py`：生成历史摘要。提供 `summarize_history(contact_id, before_timestamp)` 方法，调用 LLM 将指定时间之前的对话压缩为 300 字以内的摘要。

### 3.4 Skill 系统 (`code/src/skills/`)

- `base.py`：`BaseSkill` 抽象基类，定义 `name`、`description`、`execute(context: dict) -> dict`、`get_prompt() -> str`。
- `manager.py`：`SkillManager` 单例，提供 `register(skill)`、`get(name)`、`list_all()`、`execute(name, context)` 方法。支持通过配置文件启用/禁用。
- `builtin/reply_generator.py`：核心 Skill。执行时接收上下文（人物画像 + 历史摘要 + 近期对话 + 用户额外指令），返回 3 条回复建议（`{"suggestions": ["...", "...", "..."]}`）。
- `builtin/profile_analyzer.py`：分析对话，返回更新后的画像字段（`{"personality": {...}, "hobbies": [...], ...}`）。
- `builtin/sentiment_analyzer.py`：情感分析，返回 `{"sentiment": "positive|neutral|negative", "score": 0.8}`。
- `builtin/topic_suggester.py`：话题推荐，返回 `{"topics": ["...", "..."]}`。
- `builtin/affinity_estimator.py`：好感度估算，综合情感分析 + 互动频率，返回 `{"affinity": 45}`。

> **扩展建议**：后期可添加 `CrisisDetector` Skill（检测对话危机：冷场/踩雷），用于常开模式下自动提醒用户介入。

### 3.5 Controllers 层 (`code/src/controllers/`)

- `chat_controller.py`：唯一枢纽，继承 `QObject`，定义 `pyqtSignal` 通知 UI 更新。主要方法：
  - `select_contact(contact_id)`：切换联系人 → 加载最近 50 条消息 → 加载画像 → 通知 UI 刷新
  - `send_message(contact_id, content, sender_type)`：保存消息 → 追加到 UI → 触发后续处理
  - `load_older_messages(contact_id, before_timestamp)`：分页加载更早消息
  - `summon_agent(contact_id, extra_instruction=None)`：收集上下文 → 调用 `ReplyGenerator` Skill → 返回 3 条建议
  - `update_strategy(contact_id, user_thought)`：打开策略调整弹窗 → 携带用户想法重新调用 Skill
  - `toggle_mode(mode)`：切换静默/常开模式
  - `view_profile(contact_id)`：弹出 ProfileDialog

### 3.6 UI 层 (`code/src/ui/`)

- `main_window.py`：QMainWindow，左右布局（QSplitter），包含联系人列表和右侧聊天区域 + 输入面板。初始化时创建 Controller 并注入各子控件。
- `contact_list.py`：QListWidget，显示联系人项（头像 + 名称 + 最后消息预览）。底部有"+"按钮新建联系人。
- `chat_area.py`：QListWidget，显示消息气泡。监听滚动条到顶部时发射 `load_older_signal`。
- `input_panel.py`：QTextEdit（多行输入）+ QComboBox（切换"我/对方"）+ 按钮组（发送、召唤、策略调整）。
- `profile_dialog.py`：QDialog，显示人物基本信息、性格爱好、好感度进度条。包含"刷新画像"按钮。
- `strategy_dialog.py`：QDialog（带边框样式），包含 QTextEdit 供用户输入想法，确认后触发 Controller 重新生成策略。
- `styles.py`：集中管理 QSS。定义颜色常量（如 `#07C160` 微信绿）、字体、圆角、气泡样式等。
- `widgets/avatar_widget.py`：QLabel 圆形头像（使用 QPainter 裁剪）。
- `widgets/message_bubble.py`：QFrame，根据 sender_type 切换背景色和对齐方式，支持文本/图片/表情内容。
- `widgets/affinity_bar.py`：QProgressBar，范围 -100~100，显示好感度数值。

### 3.7 Utils 层 (`code/src/utils/`)

- `config.py`：加载/保存 `config/config.yaml`。提供 `load_config()`、`save_config(config)`、`get_llm_config()`、`get_ui_config()` 函数。首次启动若无配置文件，自动复制 `code/config/config.yaml.template` 到 `config/config.yaml` 并弹出配置引导。
- `logger.py`：loguru 配置，输出到 `logs/love_strategist.log`，按天轮转。
- `thread_utils.py`：`AsyncWorker` 类（继承 `QObject`，带 `finished`/`error` 信号），封装在 `QThread` 中执行耗时任务（LLM 调用、数据库批量查询）。

## 4. 关键架构决策

| 决策点         | 选择                                     | 理由                                                         |
| :------------- | :--------------------------------------- | :----------------------------------------------------------- |
| 架构模式       | MVP 变体（UI ↔ Controller ↔ Service）    | PyQt5 信号/槽天然适合事件驱动；Controller 作为唯一枢纽，避免 UI 直接调用 Service |
| GUI 框架       | PyQt5                                    | 成熟稳定，跨平台，QSS 支持自定义样式，仿真微信 UI 可行       |
| ORM / 数据访问 | 原生 `sqlite3` + 手写 DAO                | 避免引入 SQLAlchemy 额外依赖，项目规模小，SQL 简单可控       |
| 数据库         | SQLite 单文件                            | 零配置，纯本地，无需安装服务，备份简单                       |
| LLM API        | OpenAI 兼容接口（可配置 base_url）       | 支持 OpenAI、Claude（通过代理）、国产模型（DeepSeek/智谱等），用户自由选择 |
| Token 优化     | 三层上下文（画像 + 近期窗口 + 历史摘要） | 避免将全部聊天记录送入 LLM，节省 Token 成本                  |
| 异步处理       | QThread + 信号/槽                        | 不阻塞 UI 线程，LLM 调用和数据库查询在后台执行               |
| Skill 系统     | 可插拔架构（继承 BaseSkill + 注册）      | 方便扩展新 Skill，核心逻辑与框架解耦                         |
| 配置管理       | YAML 文件                                | 人类可读，易于修改                                           |
| 日志           | loguru                                   | 配置简单，自动轮转，支持结构化日志                           |
| 依赖注入       | 手动构造函数注入（在 main.py 中组装）    | 不引入 DI 框架，保持简单透明                                 |
| 密码/安全      | 无本地密码                               | 纯本地应用，无用户登录模块                                   |

## 7. 风险与注意事项

1. **Token 成本控制**：反复加载完整聊天记录会快速消耗 Token。必须严格使用三层上下文策略（画像 + 近期窗口 + 历史摘要），并在 UI 中显示每次调用的 Token 消耗估算。详见 `docs/Token优化方案.md`。
2. **异步 UI 响应**：LLM API 调用耗时可能 3-10 秒，必须在 QThread 中执行，否则 UI 会卡顿。所有 `llm_service` 调用需通过 `AsyncWorker` 封装。
3. **SQLite 并发写入**：UI 线程和后台线程可能同时写入数据库（如用户发送消息 + 后台画像更新）。所有写操作需使用 `threading.Lock` 保护，或使用 SQLite 的 `WAL` 模式。
4. **配置安全**：API Key 存储在 `config/config.yaml` 中，该文件已加入 `.gitignore`，避免误提交。首次启动应提示用户填写。
5. **上下文窗口限制**：不同模型上下文窗口不同（GPT-3.5 为 16K，GPT-4 为 128K）。需根据配置的模型动态调整近期窗口大小，确保总 Token 不超过模型限制。
6. **回复风格一致性**：需在系统提示词中明确约束回复风格（00 后、碎片化、不油腻），并在 Skill 中持续强化。风格 Prompt 模板见 `docs/回复风格控制.md`。
7. **人物画像准确性**：画像由 LLM 从聊天记录中提炼，可能存在偏差。允许用户通过 ProfileDialog 手动编辑画像字段，并提供"反馈"机制修正。
8. **聊天记录图片处理**：图片和表情包仅做文本化描述（用户手动输入文字描述），不存储图片二进制数据，减轻存储压力。
