## 临时文件管理规范

所有工程代码在 `code/` 目录下。临时文件统一放在项目根目录的 `temp/` 下：

| 类型         | 存放位置            | 说明       |
| :----------- | :------------------ | :--------- |
| 开发测试脚本 | `temp/test_*.py`    | 功能验证用 |
| 临时数据导出 | `temp/export/`      | 调试用数据 |
| 截图/录屏    | `temp/screenshots/` | 文档用     |
| 中间生成文件 | `temp/cache/`       | 如模型缓存 |

开发完成后，运行清理脚本移除 `temp/` 下所有内容：

```bash
cd code && python scripts/clean_temp.py
```

## 性能与体验建议

1. **消息加载**：使用分页加载（每次 50 条），避免一次性加载全部。
2. **LLM 调用**：使用 `QThread` 异步执行，界面不卡顿。参见 `code/src/utils/thread_utils.py`。
3. **数据库索引**：确保 `messages(contact_id, timestamp)` 有复合索引。
4. **启动速度**：懒加载——启动时只显示联系人列表，点击后再加载聊天记录。
5. **SQLite WAL 模式**：建议开启 WAL 模式提升并发读性能：`PRAGMA journal_mode=WAL;`

## Token 使用注意事项

1. 绝不将完整聊天记录全部送入 LLM（详见 `docs/Token优化方案.md`）。
2. 每次 LLM 调用后记录 token 消耗到 `skill_logs` 表，便于用户了解用量。
3. 根据模型上下文窗口动态调整近期对话窗口大小。
4. 系统提示词中固定回复风格约束，避免每次重复注入。
